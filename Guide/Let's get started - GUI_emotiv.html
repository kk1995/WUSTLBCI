Let's get started

Thanks to improvements in wireless technology and electronics, we can afford to extract low cost EEG data without worrying about bulky, long wires!!
However, we should first check our inventory before we get serious.
First, we need an EEG device, and a computer (or a computing machine that is capable of running Matlab).
One of the most prevalent commercialized EEG device is Emotiv EPOCTM.
There are many other commercial device, but our GUI is compatible only with Emotiv device. (future update might include MUSETM)
 

(Above is my lab's Emotiv set)
You will see a EEG headset, felt pads and a dongle. (a charger is not included in the picture)

EEG headset: This will measure voltage and send the data to computer through dongle
Dongle: You might think it is a thumb drive. This little USB porter is a linchpin of data transfer between the headset and the computing machine. (A.K.A. never lose it)
Felt pads: There are total of 16. They need to be moisturized by saline, like eye drops. They need to be properly hydrated to have a good conductance.

All of these combined is no more than $400. If you buy a license, the total cost will be around $700. You should have a license that allows you to have a full access to raw EEG data. (It is very important that you have a fully access to raw EEG, because you might otherwise have to hack the device, which I think not worthy of trying for multiple reasons.)

After you have all of the hardware, we need to install the software. 
This is the inventory of softwares that you need to successfully run Emotiv and GUI files:
Matlab 32bits* (later than 2012 version)
edk.dll
edk.h
GUI_Emotiv_version_X_X_X.m
filcker_6.exe
Emotiv SDK**
Chronux***
eeglab***
EmotivEEG.m
* Matlab 32bits is required, not 64bits. There is a good reason for using 32bits, and I will explain it later.
** Emotiv SDK is required if you want impedence test, which is used for 'set up your headset' section. If you have other machine that has SDK or you don't really care about the impedence, don't bother installing the SDK. If you have all the other files, GUI should work without SDK.
*** go to 'download' section for the links.

This tutorial will take you through the process of installation from the scratch. If you have edk.dll and edk.h files, just skip to setp 3.

1. Download, install SDK
When you order the license, you will receive a CD that contains SDK installation file. For my lab's case, we have a education edition, which allows full access to raw EEG.


The set up should look like this. Just go along with the instruction.
Once you are done with the instruction, you should have these icons:
(of course you will have different application installed depending on what license you have)

You can try out different app they have included. But the most important application is 'EmotivControlPanel.exe'
That executable is what you run for a impedance test.    >>> Please look at 'Setup your headset' for details

2. Get core files from SDK
After the proper SDK installation, you should have directory 'C:\Program Files(x86)\Emotiv Education Edition SDK v2.0.0.20' or something like this depending on your installation option.

go to'~\Emotiv Education Edition SDK v2.0.0.20\doc\examples_Matlab'
You will have sample matlab codes such as 'affectiv' or 'eeglogger'.
These matlab files are using c library from edk.dll and edk.h to give order to emotiv device (emotiv does not speak in Matlab so we sort of need to translate our program language to C language)

Instead of using 'affectiv' or 'eeglogger', we will use 'EmotivEEG' from a user community.
This file is free to download at: http://www.mathworks.com/matlabcentral/fileexchange/36111-emotiveeg-headset-toolbox
'EmotivEEG' is also dependent on edk.dll and edk.h for C library, so we need those files in the matlab directory whenever we run the file.

3. Set up environment for GUI file
First, make a folder at directory: ~\Documents\MATLAB\(your folder name)
*From now on, all files will be saved in MATLAB folder under Documents. This is where all the *.m files resides. (Of course you can move files to other directory, but have to set the path for Matlab. You will find it most convenient for you if you save all .m files in the MATLAB folder).
Copy 'edk.h' and 'edk.dll' to your folder.
Download and copy 'EmotivEEG.m', 'GUI_Emotiv_version_1_0_7.m' and 'flicker_6.exe' to your folder.
You should see something like this:

(I have other files, but ignore those)

Now you need to compile the C library to let your matlab to recognize and 'translate' your codes for Emotiv device.
Open Matlab (32bits) and type: mex -setup
You should see:
You should see Lcc-win 32. This compiler is REQUIRED for EmotivEEG to call functions in edk.dll file. This compiler is only available in 32bits matlab.
Chose Lcc-win 32. NEVER chose Microsoft SDK compiler.
If you don't have 32 bits matlab, install it.

Press y to confirm your choice and you will see:
Your matlab is now equipped with proper compiler to deal with 'edk.h' and edk.dll' file.

Now, you need toolboxes for matlab codes. Please download Chronux and EEGlab.
Chronux: http://chronux.org/index.php/files
EEGlab: http://sccn.ucsd.edu/eeglab/install.html
After you downloaded the files, you need to set them in matlab path.
Press 'set path' as shown with red eclipse on the picture.
and select the folder that contains Chronux and EEGlab. Don't forget to click ' add with subfolder'

Finally, open 'GUI_Emotiv_version_1_0_7.m'
Press 'start'. If you see a screen like this then you have set up correctly:


Now you are ready to start collecting interesting EEG data!!

For mac users: I have not tried this on Mac yet, but please try the same step as windows users. If there are problems, please contact
